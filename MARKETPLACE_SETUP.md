# Marketplace setup

This file lists every step needed to turn the marketplace code on. Without these, `/sell`, `/marketplace`, `/listing`, and `/account` will load but nothing will save or pay.

## 1. Supabase project

Create a new Supabase project (or reuse one — the marketplace lives in its own tables so it won't clash with existing data).

In the SQL editor, paste and run `supabase/migrations/0001_marketplace_init.sql`. That creates:

- `profiles` (auto-populated on signup)
- `listings`, `listing_images`, `listing_payments`
- a public `public_listings` view used by `/marketplace`
- a public storage bucket `listing-images` with policies that let signed-in users upload only into their own folder

In **Authentication → Email** templates, set the **Magic Link** subject and body to match the MowRight brand (the default is fine for testing). Make sure **Site URL** is `https://mowright.co.uk` and **Additional redirect URLs** includes `https://mowright.co.uk/sell` and `https://mowright.co.uk/account`.

Grab from **Project settings → API**:

- Project URL
- `anon` public key
- `service_role` secret key

## 2. Stripe

In Stripe Dashboard:

1. Switch to live mode when you're ready, or stay on test mode while you try it.
2. **Developers → API keys**: copy the secret key (`sk_live_…` or `sk_test_…`).
3. **Developers → Webhooks → Add endpoint**:
   - URL: `https://mowright.co.uk/api/stripe-webhook`
   - Events to send: `checkout.session.completed` (you can also enable `charge.refunded` if you ever want to handle refunds later).
   - After saving, copy the **Signing secret** (`whsec_…`).

The listing fee is set in code via env vars (default £2.99). To change the price, update `LISTING_FEE_PENCE` on Vercel and the `<meta name="mw-listing-fee-pence">` tag inside `sell.html`.

## 3. Vercel environment variables

In Vercel → Project → Settings → Environment Variables, add for **Production** (and Preview if you want):

| Name | Value |
| --- | --- |
| `SUPABASE_URL` | from step 1 |
| `SUPABASE_ANON_KEY` | from step 1 |
| `SUPABASE_SERVICE_ROLE` | from step 1 (server-only — do not expose in HTML) |
| `STRIPE_SECRET_KEY` | from step 2 |
| `STRIPE_WEBHOOK_SECRET` | from step 2 |
| `LISTING_FEE_PENCE` | `299` (optional override) |
| `LISTING_DURATION_DAYS` | `30` (optional override) |
| `SITE_URL` | `https://mowright.co.uk` |

## 4. Wire the public Supabase keys into the HTML

The four marketplace HTML files contain placeholders:

```html
<meta name="mw-supabase-url" content="__SUPABASE_URL__"/>
<meta name="mw-supabase-anon-key" content="__SUPABASE_ANON_KEY__"/>
```

Replace `__SUPABASE_URL__` with your project URL and `__SUPABASE_ANON_KEY__` with the anon key in each of:

- `sell.html`
- `marketplace.html`
- `listing.html`
- `account.html`

These keys are **public** (they're the anon key — Supabase RLS enforces access control), so it's fine to bake them into the HTML.

## 5. Deploy + test

Commit and push. Vercel will install `stripe` and `@supabase/supabase-js` automatically (they're in `package.json`) for the serverless functions, and the static pages will pick up the meta-tag config.

Then:

1. Visit `/sell` — confirm the magic-link email lands.
2. Create a listing with one photo, click "Continue to payment".
3. Pay with Stripe's test card `4242 4242 4242 4242` (any future expiry, any CVC).
4. Confirm Stripe Dashboard → Events shows the `checkout.session.completed` event delivered with a 200 response.
5. Open `/account` — the listing should be marked **live**.
6. Open `/marketplace` — the listing appears in the grid.
7. Open `/listing?id=…` — click each social share button to confirm pre-fill text is sensible.

## 6. Optional: cron job to expire stale listings

The `public_listings` view already hides listings where `expires_at < now()`, so the marketplace stops showing them automatically — no cron strictly required.

If you also want their **status** column to flip to `'expired'` (so the seller's `/account` page reflects it), add a Vercel Cron entry to `vercel.json`:

```jsonc
{
  "crons": [{ "path": "/api/expire-listings", "schedule": "0 3 * * *" }]
}
```

…and create `api/expire-listings.js` that runs `update listings set status='expired' where status='live' and expires_at < now()`. Not included by default to keep the surface area small.

## 7. Existing-page nav refresh

The marketplace adds two nav links (`Marketplace`, `Sell`). They are wired into `index.html`, `sell.html`, `marketplace.html`, `listing.html`, and `account.html` directly. For all other pages (about, contact, blog, category pages, brand pages, individual mower pages, etc.), the nav is generated by `scripts/build.mjs` — run `npm run build` once after deploy to refresh them.
